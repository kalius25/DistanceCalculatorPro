ADR-027 — Rich Exception Model
ADR ID         : ADR-027

Title          : Rich Exception Model

Status         : Approved

Supersedes     : None

Revision        : 1.0.0
Context

DistanceCalculatorPro sẽ phát triển thêm nhiều Provider, Engine, Parser và Service trong các Sprint tiếp theo.

Hệ thống cần một mô hình Exception thống nhất để:

truyền đạt lỗi giữa các Layer;
hỗ trợ Logging;
hỗ trợ Retry;
hỗ trợ Debug;
hỗ trợ ErrorCode chuẩn hóa.

Exception hiện tại chỉ lưu message, chưa đủ thông tin cho các nhu cầu trên.

Decision

Toàn bộ Exception của dự án phải kế thừa từ một Base Exception duy nhất:

DistanceCalculatorError

Base Exception chuẩn sẽ hỗ trợ tối thiểu các thuộc tính:

message
error_code
cause
context

Các Exception chuyên biệt như:

EngineException
ParserException
ProviderException

không tự quản lý các thuộc tính trên mà kế thừa từ Base Exception.

Backward Compatibility

API cũ vẫn phải hoạt động.

Ví dụ:

raise ParserException("Invalid HTML")

vẫn hợp lệ.

Đồng thời hỗ trợ API mở rộng:

raise ParserException(
    "Invalid HTML",
    error_code=ErrorCode.PARSER_ERROR,
    cause=e,
    context={
        "provider": "Google",
        "url": current_url,
    },
)

Không được yêu cầu sửa mã nguồn cũ.

Alternatives Considered
Alternative 1

Mỗi Exception tự bổ sung thuộc tính riêng.

Không được chọn.

Nhược điểm:

Trùng lặp mã nguồn.
Không thống nhất.
Khó Logging.
Khó Retry.
Khó mở rộng.
Alternative 2

Sử dụng Rich Base Exception.

Được chọn.

Ưu điểm:

Một mô hình thống nhất.
Dễ mở rộng.
Dễ bảo trì.
Hỗ trợ Logging và Retry tốt hơn.
Consequences

Sau ADR này:

Mọi Exception có cùng cấu trúc.
ErrorCode được chuẩn hóa.
Logging có thể lấy đầy đủ ngữ cảnh.
Retry có thể dựa trên ErrorCode.
Các Layer trao đổi lỗi theo cùng một mô hình.
